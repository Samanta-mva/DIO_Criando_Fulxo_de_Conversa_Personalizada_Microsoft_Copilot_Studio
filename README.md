## Desafio DIO

# Criando um Fluxo de Conversa Personalizado

O desafio do Bootcamp da Suzano na plataforma da DIO propõe criar um fluxo de conversa personalizado para o agente da Microsoft Copilot Studio e criar um resumo do que foi aprendido e publicar no GitHub.



### Resumo

Criar um fluxo de conversa personalizada no Microsoft Copilot Studio significa estipular como o agente vai conduzir a conversa, com tópicos, gatilhos, perguntas, respostas e ramificações para cada cenário. A ideia é transformar um atendimento genérico em uma experiência guiada, seguindo uma lógica de raciocínio, utilizando variáveis e respostas mais contextualizadas.



#### Como funciona:

Você ajusta o gatilho, adiciona frases de ativação, cria perguntas ao usuário, define condições e conecta ações, bases de conhecimento ou fluxos externos quando necessário.



#### Elementos principais:

Tópicos: organizam cada assunto ou intenção da conversa.



Gatilhos: definem quando o tópico deve ser iniciado, seja por frases do usuário ou por escolha do agente ou uma chamada por ação.



Variáveis: guardam informações como nome, e-mail ou preferência para personalizar a resposta.



Ramificações: criam caminhos diferentes conforme a resposta do usuário.



Fallback: trata perguntas que o agente não entendeu e evita a conversa “quebrar”.



#### Vantagens:

Esse tipo de fluxo deixa o agente mais claro, previsível e útil, porque cada etapa é pensada para um objetivo específico. Também ajuda a combinar linguagem natural com regras de negócio, automações e respostas gerativas quando fizer sentido.



### Conclusão Prática:

Na prática, você:

- Cria o tópico;

- Define o gatilho;

- Faz perguntas ao usuário;

- Usa variáveis para guardar dados;

- Cria condições para cada resposta;

- Testa e publica o fluxo.
